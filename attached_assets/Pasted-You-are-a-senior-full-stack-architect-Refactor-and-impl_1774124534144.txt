## 3. ROLE PERMISSIONS

### SECRETARIA
ALLOW:
- patients.read
- appointments.read
- appointments.create
- appointments.update

DENY:
- medical.write
- financial.*
- settings.manage
- users.manage

---

### PROFISSIONAL
ALLOW:
- all from SECRETARIA +
- medical.read
- medical.write
- appointments.read

DENY:
- settings.manage
- users.manage
- financial.write

---

### ADMIN
ALLOW:
- ALL permissions (full access)

---

## 4. SPECIAL RULE

A user can have MULTIPLE ROLES.

Example:
- PROFISSIONAL + ADMIN (autonomous professional)

## 5. JWT UPDATE

When generating JWT include:

{
  userId,
  roles: ["ADMIN", "PROFISSIONAL"]
}

## 6. BACKEND (Express)

Create middleware:

requirePermission(permissionKey: string)

Behavior:
- Read user roles from JWT
- Fetch permissions from roles_permissions
- Allow or deny request

Usage example:

app.get("/api/patients", requirePermission("patients.read"), handler)
