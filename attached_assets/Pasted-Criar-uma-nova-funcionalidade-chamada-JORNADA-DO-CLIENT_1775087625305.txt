Criar uma nova funcionalidade chamada:

JORNADA DO CLIENTE

Essa funcionalidade deve ser integrada ao prontuário do paciente e representar visualmente toda a jornada clínica desde o cadastro até a alta.

O agente deve analisar o projeto existente e implementar a solução seguindo automaticamente:

padrões de código já utilizados
arquitetura atual do sistema
regras de segurança
estrutura de permissões
design system existente
convenções de banco de dados
responsividade já aplicada no sistema

Não criar soluções isoladas.
Sempre reutilizar padrões existentes.

OBJETIVO

Criar uma nova aba no prontuário chamada:

Jornada do Cliente

Essa aba deve exibir uma linha do tempo clínica (stepper) mostrando todas as etapas do atendimento do paciente, em ordem sequencial.

Essa jornada representa o fluxo real do atendimento dentro da clínica.

Ela será usada por:

Secretária
Profissional
Administrador
COMPORTAMENTO GERAL

A jornada deve sempre existir para todo paciente.

Se um paciente não tiver jornada:

O sistema deve criar automaticamente todas as etapas.

Nunca remover etapas.

Sempre manter a sequência obrigatória.

Nunca permitir pular etapas.

ETAPAS DA JORNADA

A jornada deve conter exatamente estas etapas, nesta ordem:

1 Cadastro
2 Anamnese
3 Avaliação
4 Plano de Tratamento
5 Procedimentos / Pacotes
6 Agendamento
7 Tratamento em andamento
8 Alta

A ordem é obrigatória.

STATUS DAS ETAPAS

Cada etapa deve possuir um status.

Status possíveis:

pending
in_progress
completed
cancelled

COMPORTAMENTO VISUAL

A aba deve exibir:

Uma linha do tempo vertical.

Cada etapa deve mostrar:

Nome da etapa
Status
Data de início
Data de conclusão
Responsável
Observações
CORES DOS STATUS

pending:

cinza

in_progress:

amarelo

completed:

verde

cancelled:

vermelho

Usar o padrão visual já existente no sistema.

INTERAÇÕES

Cada etapa deve permitir:

Avançar etapa
Editar
Cancelar etapa

REGRA DE AVANÇO

Ao avançar uma etapa:

A etapa atual deve ser marcada como:

completed

A próxima etapa deve ser marcada como:

in_progress

EXEMPLO

Se:

Avaliação está em andamento

Ao clicar:

Avançar

O sistema deve:

Avaliação → completed
Plano → in_progress

ATUALIZAÇÃO AUTOMÁTICA

O sistema deve atualizar a jornada automaticamente com base nas ações já existentes no prontuário.

Quando:

Paciente for criado:

Cadastro → completed

Quando:

Anamnese for salva:

Anamnese → completed

Quando:

Avaliação for criada:

Avaliação → completed

Quando:

Plano de tratamento for criado:

Plano → completed

Quando:

Pacote ou procedimento for vendido:

Procedimentos → completed

Quando:

Primeiro agendamento for criado:

Agendamento → completed

Quando:

Primeira sessão for concluída:

Tratamento → in_progress

Quando:

Alta for registrada:

Alta → completed

REGRA IMPORTANTE

Não permitir:

Concluir uma etapa se a etapa anterior estiver pendente.

Plano:

completed

Avaliação:
